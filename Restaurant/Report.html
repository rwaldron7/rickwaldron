Report

The original website for the Garden Indian Restaurant contained all of the necessary information, however it lacked two main features:

  1. It was not a modern website. The design was outdated and it was not optimised for mobile devices.
  2. It lacked the ability to interact directly through the website. This includes the ability to make a booking or to order take away.

This new design is aimed at addressing these two main issues.

In order to update the website to look and feel modern, the starting point was the header at the top. This needed to be simple so as to make it easy for the user to navigate the website. A new, modern font was selected from Google Fonts and a dark red was applied to the header and footer of the website. This colour contrasts well on white and looks clean and modern. The header, including navigation links, was designed to change shape as the screen size decreases.

In the center of the page it was decided to place a new picture as the background. The original pictures on the website were low quality and did not look professional. A picture (see attribution below) was found that looked very appealing and helped draw the users attention to the main product of the restaurant: food. The main text of each page was placed on top of this picture with white text, and because the text would often overflow the ability to scroll down was added to the element.

It was decided that such information as opening hours, phone number and address were important enough to add to every page of the website. These were added to the footer on each page, which was styled in a similar way to the header. This creates symmery in the webpage, and draws the users attention towards the main part of the page while also allowing easy access to important restaurant details.

The lunch menu and dinner menu were updated by removing borders except for a line under each subheading. Original pictures were added to an image carousel on a gallery page that the user can click through at their own pace.

The interactive elements of the website include search, bookings and a take away app:

The search function is accessible by a search icon at the top of each page. Clicking on this produces an input box where the user can type in their request. Pressing enter then opens a new page with search results, and page which contains any combination of the search query.

The bookings form has been updated from the original by adding a form that the guest can add their details, as well as preferred date and time. It was decided to place each input on a separate line to keep with the clean look of the rest of the website. Placeholders were provided for the last two inputs to give guests an idea of the type of input expected.

The take away app is the most significant new component of this website. It extends the take away menu by the addition of an "Add" button next to each menu item. Upon clicking this button, diners are asked for a quantity. They can do this for as many items as they choose, and then can press "Add to Cart" which will redirect them to the Shopping Cart page. Here, they will be presented with their current selections, quantity and total price. If they are happy with their order, they can enter their first name and mobile number and press "Submit Order". Otherwise, they can press "Clear Shopping Cart" to start again. It was decided to keep the details as short as possible here to make the ordering process as quick and smooth as possible.

The take away app uses local storage to save the shopping cart to the browser. This means that if the user clicks out of the shopping cart and returns later, they will still see the items they previously selected.

For mobile users, the website has been designed to change for smaller screen sizes. The background image for the main text is not appropriate for smaller screen sizes, so is replaced by a black background for these devices. Furthermore, the navigation links are replaced by a hamburger menu that, when clicked, diplays the links down the page.
